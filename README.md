# ТЗ

## Цели проекта

Обязательные:

1. Написать одностраничное приложение, содержащее список задач и способы управления ими
2. Добавить календарь для планирования задач по дате и времени
3. Создать удобный, интуитивно понятный интерфейс

## Описание работы

Приложение состоит из логических модулей:

1. Модуль аутентификации и авторизации пользователя
2. Модуль меню
3. Модуль работы с пулом 
4. Модуль создания и настройки списка
5. Модуль создания и настройки задачи
6. Модуль календаря
7. Модуль настроек

## Более подробное описание работы модулей

### 1.1 Авторизация

Предполагается окно регистрации и входа в систему. Нужно, чтобы для каждого пользователя был уникальный to-do лист.

Можно использовать JWT токены для аутентификации и авторизации

### 1.2 Регистрация в системе

Для регистрации нужно заполнить следующие поля:

1. Логин - символы латинского алфавита и цифры
2. E-mail - строка формата почтового адреса
3. Пароль - допустимы символы латинского алфавита, цифры, спецсимволы !@#$%^&*
4. Подтверждение пароля - сравнивается с паролем

Данные отправляются в базу данных (ну, видимо это будет просто массив, так как бэка не предполагается)

### 1.3 Вход в систему

Для входа в систему нужно заполнить следующие поля:

1. Логин или почта
2. Пароль

Они сопоставятся с данными из БД (списка), и если данные верны, доступ будет разрешен

### 2.1 Меню приложения

В меню находится панель-сайдбар с разделами приложения. На ней представлена вкладка списков, вкладка календаря и вкладка настроек.

На вкладке со списками представлен пул списков и содержимое выбранного списка

На вкладке календаря находится календарь и сайдбар с выбором представления (день, неделя, месяц)

На вкладке настроек расположено меню настроек.

### 3.1 Пул списков

Для каждого профиля предполагается пул со списками. Первым идет список “Мой день”,

затем “Нераспределенные задачи”, затем все пользовательские списки. В пуле можно создавать и редактировать списки

### 4.1 Создание списков

При создании списка можно выбрать, будет ли он автоматическим или ручного составления. Списку можно присвоить уникальный эмоджи, который будет отображаться слева от названия списка в пуле, это поможет быстро различать списки. 

Есть неудаляемый общий список “Нераспределенные задачи”, в который будут попадать задачи, не привязанные ни какому списку, а также задачи, созданные в автоматических списках

Есть неудаляемый автоматический список “Мой день”. В этот список отправляются все задачи с сегодняшней датой.

В списках можно настроить сортировку по параметру.

Порядок задач в списке задается пользователем и фиксированный, либо задается сортировкой. Всегда при сбросе сортировки возвращается к порядку, заданному пользователем. Перетаскиванием (drag-and-drop) задач пользователь может менять их положение в списке

### 4.2 Автоматический список

Списки, которые будут формироваться по определенным тегам, например по важности, по тегу, который задал пользователь, по дате. Формирование будет происходить автоматически по существующим задачам, пользователь сможет сам настроить отображение этих списков, дать им название. 

При добавлении задачи (по нажатию кнопки ”+”) она будет создана в списке “Нераспределенные задачи” и автоматически помечена тегом автоматического списка, в котором она создавалась

В списке могут быть ТОЛЬКО задачи, помеченные тегом этого списка

### 4.3 Ручной список

Все задачи добавляются вручную пользователем по нажатию кнопки “+”, тега задаче не присваивается. Список может содержать абсолютно разные задачи, объединяет их только принадлежность списку 

### 4.4 Завершенные задачи

Завершенные задачи уходят в нижний раздел списка “Выполнено”

Также есть общий список со всеми завершенными задачами.

### 5.1 Создание задач

В каждом списке можно создавать задачи, нажав кнопку со знаком “Создать задачу”. У задачи могут быть следующие задаваемые параметры:

1. Название задачи
2. Дата и время начала, дата и время окончания либо же повторяющееся событие
3. Описание
4. Эмоджи-значок
5. Геометка (???сложности реализации???)
6. Файл (документ, картинка)
7. Принадлежность списку 

Задачи можно удалять, редактировать, переносить из списка в список.

У задачи может быть список подзадач, каждую можно именовать (и ничего более) при выполнении их всех основная задача отмечается выполненной

При создании задачи автоматически определяется принадлежность списку, но в процессе создания задачи можно его изменить.

### 5.2 Временная настройка задачи

Задаче можно задать либо дату, либо дату+время, либо дату+время начала и дату+время конца события. По этим данным в календаре отмечается событие.

Если задать только дату, то задача отметится на весь день. Если указать дату+время, то задача отметится в календаре как точка старта. Если указать дату+время начала и дату+время конца, то задача начнет занимать в календаре некоторый объем от начала и до конца задачи. 

### 5.3 Повторы

Можно настроить повтор ежедневно, еженедельно, ежемесячно, ежегодно.

Можно указывать только при выборе даты, от нее будут отсчитываться повторы.

Настройки времени не влияют на повторы.

### 6.1 Календарь

 Календарь будет отображать все временные метки.

У календаря будет несколько представлений: день, неделя, месяц.

В представлении дня:

1. День представлен вертикальной почасовой временной шкалой, может масштабироваться
2. События с меткой по дате будут отображаться сверху над шкалой
3. События с меткой по времени будут отображаться в момент времени (проблемы UI дизайна)
4. События с длительностью будут заполнять пространство от начала до конца задачи

В представлении недели:

Все так же, как и с представлением дня, но:

1. На экране отображена вся неделя, поэтому каждый день сильно уже
2. Шрифты в задачах меньше, чтобы максимум информации поместилось на минимуме пространства
3. Сверху отображены названия дней недели и задачи, определенные на дату

В представлении месяца:

1. Будет представлен календарный месяц, стрелками вправо и влево он будет меняться. На каждом дне будут отображаться только те задачи, которые определены на дату.

### 6.2 Перенос задач в календарь

Все задачи, имеющие временную метку, автоматически будут добавляться в календарь. Цвет мероприятия в календаре будет синхронизирован с цветовой меткой задачи

### 6.3 Создание задач в календаре

В календаре возможно создавать задачи. При нажатии на участок, ограниченный часом, появляется окно создания задачи, в котором задана длительность от начала до конца часа. Если зажать кнопку мыши и повести ее вдоль линии таймлайна, возникнет выделение, с помощью которого можно регулировать длительность события

### 7.1 Раздел настроек

В настройках должны быть следующие параметры:

1. Настройка цветовой схемы
2. Выбор языка
3. Смена пароля
4. Формат даты и времени

## Требования к приложению

### 1. Требования к интерфейсу

Интерфейс должен быть простым и понятным, названия вкладок и функций должны передавать максимум смысла, цветовые контрасты и значки должны помогать ориентироваться в структуре приложения.

Уровень вложенности различных меню должен быть не больше 2х (в крайнем случае 3х), все функции должны быть логически разделены на разделы и не теряться в приложении

### 2. Требования к дизайну

Минималистичный и легко считываемый дизайн.

### 3. Требования к функционалу

Приложение должно иметь:

1. Функционал работы с задачами
2. Функционал работы с календарем
3. Функционал работы со списками

## Стек технологий

Для разработки приложения используется фреймворк Angular, язык TypeScript
